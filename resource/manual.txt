
  이 프로그램은 42 Python Module 과제 진행을 돕기 위한 보조 쉘 스크립트입니다.

  과제 폴더 및 제출 파일 자동 생성, flake8/mypy 검사와 간단한 실행 테스트,
  각 과제 수행 팁 등의 기능을 제공합니다.

  42 NEXT pythopn module ver. 3.0 기준으로 제작되었습니다.

  주 사용 쉘의 설정파일(예 : /.bashrc)에

  alias pymo="$HOME/workspace/pymo/start.sh"

  같은 예시처럼 별칭 등록을 하여
  본 헬퍼 실행 명령어를 추가하는 것을 권장합니다.

  첫 실행 후 4번 헬퍼 설정을 통해 과제를 수행할 루트 디렉토리 및
  생성될 폴더의 서식 등 본인이 원하는 설정을 하신 후 사용하는 것을 권장합니다.

  메뉴 설명

  1. 디렉토리/파일 생성

    등록된 module 번호에 맞춰 프로젝트 루트에 다음 형식의 폴더를 생성합니다.

      python_module_00
      python_module_01
      python_module_02
      ...

    각 module에 등록된 파일 목록을 기준으로 필요한 폴더와 파일을 생성합니다.
    이미 존재하는 파일은 덮어쓰지 않고 건너뜁니다.

    새로 생성된 모든 .py 파일에는 chmod +x 실행 권한을 부여합니다.
    파일 내부 상단에 #!/usr/bin/env python3 쉬뱅을 추가하여
    ./파일명.py 명령어 만으로 코드 실행이 가능합니다.

	만약 모듈 별 폴더명 서식을 바꾸고 싶다면 config.env에 있는
	MODULE_DIR_NAME 값을 지정하면 됩니다.
	모듈 별로 폴더명 끝에 00~10의 숫자가 붙는 것은 고정입니다.

	만약 폴더명 서식을 변경할 경우 기존에 작업했던 모듈과제 폴더명도
	서식에 맞게 수정해야 헬퍼에서 인식이 됩니다.

	(MODULE_DIR_NAME="module" 면 moudule00 이런 폴더명으로 생성 및 인식.)

  2. 정적 검사 및 실행 테스트

    선택한 module 폴더에서 다음 순서로 검사를 진행합니다.

      1) 파일 존재여부 검사
      2) flake8 검사
      3) mypy --strict 검사
      4) 실행 테스트
      5) .mypy_cache 및 __pycache__ 삭제

    flake8 또는 mypy가 설치되어 있지 않으면 다음 선택지를 제공합니다.

      1. pip install --user 로 설치
      2. 해당 검사를 무시하고 계속
      3. 종료

    flake8 또는 mypy 에러가 감지되면 다음 선택지를 제공합니다.

      1. 초기화면으로 돌아가기
      2. 무시하고 실행 테스트 계속하기
      3. 종료

    패키지 설치 시 ~/.local/bin 경로를 환경변수 기준으로 사용합니다.

   flake8, mypy, glow 등의 패키지는 다음 경로에 설치됩니다.

      ~/.local/bin

    ~/.bashrc 에 아래 환경변수를 자동으로 추가합니다.

      export PATH="$HOME/.local/bin:$PATH"

    만약 zsh 등의 다른 셸을 주로 사용 중이라면
    사용하는 셸의 설정파일(~/.zshrc 등)에 동일한 내용을 직접 추가해야 합니다.

    module별 테스트 방식

      각 module은 설정된 테스트 함수에 따라 실행됩니다.

    일반 테스트 방식

      일반 module은 지정된 파일을 순서대로 전부 실행합니다.
      각 파일이 있는 폴더로 이동한 뒤 python3 파일명 형식으로 실행합니다.

      예:

        python module 01/ex0/ft_garden_intro.py

      실행 시:

        cd "python module 01/ex0"
        python3 ft_garden_intro.py

    특수 테스트 방식

      module 00은 helper_python_module/resource/00/main.py 파일을
      python module 00/main.py 위치로 임시 복사한 뒤 실행합니다.
      실행이 끝나면 복사한 main.py는 삭제됩니다.

      module 09, 10 의 경우에도 샘플 파일을 리소스에서 임시 복사한 뒤 실행,
      실행이 끝나면 복사한 파일은 삭제됩니다.

      module 08, 09 같은 가상환경 세팅이 필요한 과제의 경우,
      .venv 이름의 가상환경이 설치가 되어있지 않으면 실행되지 않습니다.

    본 헬퍼는 각 모듈 별로 고정된 방식의 실행 및 출력만 지원합니다.
    따라서, 개별 파일 테스트 및 인자 변경 후 테스트 등을 원하실 땐
    이 헬퍼를 이용하지 않고 개별 파일을 직접 실행해보셔야 합니다.

    후술할 3번 과제 팁 보기에 "00_필독" 문항이 있는 모듈이 있습니다.
    실행 테스트에 직결된 안내사항이 있으니 해당 문항이 있을 경우
    가급적 열람하신 후 실행 테스트를 진행하시는 것을 권장합니다.

  3. 문제별 팁 보기

    각 과제의 개요 및 수행 팁을 출력합니다.

    팁 메뉴에서는 module 번호를 먼저 고르고,
    해당 module에서 필요한 정보에 대한 내용을 볼 수 있습니다.

    모든 팁 문서는 markdown 파일로 제작되어 있습니다.
    따라서, glow를 설치, markdown 파일을 열람하는 방법을 권장합니다.
    glow가 없다면 설치 여부를 묻고, 설치하지 않을 경우 cat으로 출력합니다.

  glow 설치 위치

    ~/.local/bin/glow

    사용자가 개별적으로 터미널에서 glow 명령어를 직접 사용하고자 한다면
    주 사용 쉘 설정 파일에 환경변수를 추가하는 법을 찾아보는 것을 권장합니다.

  팁에서는 과제 수행을 위해 필요한 모든 기술명세를 상세히 하고 있지 않습니다.
  가급적 개인학습 중 참고 용도로 사용하시는 것을 권장합니다.

  4. 헬퍼 설정

	헬퍼 프로그램의 모듈 별 파일 및 폴더 생성, 실행테스트를 진행할
	프로젝트 루트 경로 및 생성될 폴더명 서식을 변경할 수 있습니다.

	프로젝트 루트에 폴더명 서식이 맞지 않는 과제폴더는 인식되지 않습니다.

	프로젝트 루트 변경 및 폴더명 서식 변경을 할 경우 과제 별 생성파일이
	존재하는 폴더는 자동으로 해당 모듈의 과제 폴더로 인식,
	서식대로 이름이 변경이 됩니다.
	
	폴더 인식 로직은 폴더명에 00~10 숫자가 있을 때 내부에 각 모듈과제 별
	생성해야 할 파일이 단 1개라도 존재할 때 입니다. 폴더명 인식 규칙 상
	같은 모듈 폴더를 여러개 생성 후 작업하시는 경우 유의해야 합니다.

	만약, 과제 별 파일이 존재하는 폴더가 중복 존재한다면
	폴더명 교체 작업 중 에러메세지가 나오므로 주의합니다.

	추가로 평가진행에도 본 헬퍼를 이용하는 경우, 항상 프로젝트 루트를
	변경하는 것이 어려울 수도 있습니다.

	본 헬퍼 패키지 루트경로에 base.env 파일을 추가로 생성하여 config.env와
	동일한 서식으로 작성해두면 초기화 시 base.env값 기준으로 변경됩니다.

  5. 종료

    프로그램을 종료합니다.
