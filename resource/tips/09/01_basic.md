# Module 09 개요

## Pydantic 데이터 검증

Pydantic은 Python 타입 표기를 기준으로 외부 데이터를 검사하고 변환하는 라이브러리입니다.

이번 모듈에서는 다음 개념을 순서대로 학습합니다.

- `BaseModel`을 이용한 데이터 모델 정의
- `Field`를 이용한 길이와 범위 제한
- `ValidationError`를 이용한 검증 실패 처리
- `Enum`을 이용한 선택 가능한 값 제한
- `model_validator`를 이용한 여러 필드 사이의 규칙 검사
- 다른 모델과 `list`를 이용한 중첩 구조 검증

## 일반 클래스와의 차이

일반 클래스는 전달받은 값이 타입 표기와 일치하는지 자동으로 검사하지 않습니다.

Pydantic 모델은 객체를 생성할 때 입력을 검사합니다. 입력이 조건에 맞으면 모델 인스턴스를 만들고, 조건에 맞지 않으면 `ValidationError`를 발생시킵니다.

## 과제 수행 흐름

각 exercise는 다음 순서로 접근하면 됩니다.

1. Subject에 지정된 필드 이름과 타입 확인
2. 필수 필드와 기본값이 있는 필드 구분
3. 각 필드의 길이와 숫자 범위를 `Field`로 표현
4. 여러 필드가 함께 결정하는 규칙 분리
5. 정상 데이터로 모델 생성
6. 규칙을 하나 위반한 데이터로 오류 확인
7. 출력 예시와 실행 결과 비교

한 번에 모든 규칙을 작성하기보다 기본 모델을 먼저 실행한 뒤 검증 조건을 하나씩 추가하는 편이 오류 원인을 찾기 쉽습니다.

## Pydantic v2 주의사항

이번 과제는 Pydantic 2.x 문법을 사용합니다.

이전 버전의 `@validator` 대신 Subject에서 요구하는 `@model_validator(mode="after")`를 사용합니다. 모델을 딕셔너리로 변환할 때는 v2의 `model_dump()`를 사용할 수 있습니다.
