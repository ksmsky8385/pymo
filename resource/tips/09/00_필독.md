# Module 09 필독 사항

## helper 실행 테스트 개요

Module 09는 Pydantic 2.x의 모델 정의와 데이터 검증을 다룹니다.

본 helper는 다음 순서로 실행 테스트를 진행합니다.

1. 모듈 루트에서 이름이 `env`로 끝나는 가상환경 확인
2. 가상환경이 없으면 모듈 루트에 `venv` 생성
3. 새로 만든 `venv`에 Pydantic 설치
4. helper 리소스의 `data_generator` 디렉토리 복사
5. 테스트 데이터 생성
6. ex0, ex1, ex2의 Python 파일 실행
7. 테스트 산출물 정리 여부 확인

## 가상환경 감지

helper는 모듈 루트 바로 아래에서 이름이 `env`로 끝나는 첫 번째 디렉토리를 가상환경으로 인식합니다.

```text
env
.venv
cosmic_env
venv
```

감지된 디렉토리는 다음 활성화 파일을 포함해야 합니다.

```text
<가상환경>/bin/activate
```

기존 가상환경이 감지되면 helper는 해당 환경에 Pydantic을 자동 설치하지 않습니다. 직접 다음 명령으로 설치 상태와 버전을 확인하시기 바랍니다.

```shell
source <가상환경>/bin/activate
python3 -m pip show pydantic
```

Pydantic이 없다면 다음과 같이 설치할 수 있습니다.

```shell
python3 -m pip install pydantic --no-cache-dir
```

## 새 가상환경 처리

기존 가상환경이 없으면 다음 명령과 동일한 방식으로 `venv`를 생성하고 활성화합니다.

```shell
python3 -m venv venv
source venv/bin/activate
```

새로 생성한 환경에는 Pydantic을 pip 캐시 없이 설치합니다.

```shell
python3 -m pip install pydantic --no-cache-dir
```

`--no-cache-dir`는 다운로드 캐시를 남기지 않는 옵션입니다. 가상환경에서 프로그램 실행에 필요한 실제 패키지 파일은 정상적으로 설치됩니다.

## 데이터 생성 도구

helper는 다음 리소스 디렉토리를 모듈 루트에 복사합니다.

```text
resource/module_09/data_generator/
```

이후 모듈 루트에서 다음 명령을 실행합니다.

```shell
python3 data_generator/data_exporter.py
```

정상적으로 실행되면 모듈 루트에 `generated_data/`가 생성됩니다. 생성된 JSON, CSV, Python 파일은 모델을 직접 시험할 때 참고할 수 있는 테스트 데이터입니다.

## ex 파일 실행

helper는 활성화된 가상환경에서 다음 파일을 순서대로 실행합니다.

```shell
(cd ex0 && python3 space_station.py)
(cd ex1 && python3 alien_contact.py)
(cd ex2 && python3 space_crew.py)
```

이 과정은 각 파일이 오류 없이 실행되는지 확인하는 실행 테스트입니다. Subject의 모든 필드 조건과 사용자 정의 검증 규칙을 helper가 대신 판정하지는 않습니다.

따라서 각 `main()`에서 최소한 다음 두 경우를 직접 확인하시기 바랍니다.

- 올바른 데이터로 모델 생성 성공
- 잘못된 데이터로 `ValidationError` 발생

## 정적 검사

실행 테스트 전에는 과제 경로에 flake8과 `mypy --strict` 검사가 적용됩니다.

helper는 모듈 번호와 관계없이 루트와 모든 하위 경로에 있는 다음 디렉토리를 정적 검사 대상에서 제외합니다.

```text
*env/
data_generator/
generated_data/
```

과제의 ex0, ex1, ex2 Python 파일에는 명확한 타입 표기를 작성하시기 바랍니다.

## 테스트 산출물 정리

테스트 종료 시 다음 항목의 삭제 여부를 묻습니다.

```text
감지하거나 생성한 *env/
data_generator/
generated_data/
```

입력 없이 엔터를 누르면 삭제가 기본 동작입니다. helper가 이번 테스트에서 새로 만든 `venv`뿐 아니라 감지하여 사용한 기존 가상환경도 삭제합니다.

기존 가상환경을 보존하려면 정리 질문에서 반드시 `n`을 입력하시기 바랍니다.

정리를 건너뛰려면 `n`을 입력합니다. 가상환경과 테스트 데이터는 제출 파일이 아니므로 Git에 추가하지 않도록 주의하시기 바랍니다.
