# Module 10 필독 사항

## 정적 검사 기준

Module 10 실행 테스트에서는 `mypy --strict .` 대신 `mypy .` 기준으로 검사합니다.

과제 PDF에 제시된 함수 시그니처가 `list[dict]`, `dict`, `Callable`처럼 넓은 타입을 그대로 사용하기 때문입니다. `--strict`는 이런 제네릭 타입에 더 구체적인 타입 인자를 요구하므로, 과제의 지정 프로토타입을 유지하는 방향과 충돌할 수 있습니다.

## 데이터 생성 도구 활용

과제에서 제공하는 `data_generator.py`가 모듈 폴더 하위 경로 어디에도 없으면, helper 실행 중 모듈 루트와 각 ex 폴더에 복사된 상태로 프로그램이 실행됩니다.

따라서 각 파일의 `main()` 또는 테스트용 코드에서 다음 클래스를 import해서 사용할 수 있습니다.

```python
from data_generator import FuncMageDataGenerator
```

추천 사용 방식은 제너레이터에 선언된 각 generate 메서드로 샘플 데이터를 만들고, 본인이 작성한 함수에 직접 넣어보는 것입니다.

예시:

```python
artifacts = FuncMageDataGenerator.generate_artifacts(4)
mages = FuncMageDataGenerator.generate_mages(5)
spells = FuncMageDataGenerator.generate_spells(4)
```

제너레이터는 제출 대상 파일이 아닙니다. helper가 실행 중 모듈 루트와 각 ex 폴더에 임시로 복사하고 실행 종료 후 자동으로 정리합니다.

모듈 하위 경로에 이미 `data_generator.py`가 하나라도 있으면 helper는 ex 폴더로 복사하지 않고 실행 파일만 실행합니다.
