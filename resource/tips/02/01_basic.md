# Module 02 개요

## Exception Handling & Defensive Programming

현실의 프로그램은 항상 정상적인 입력만 받지 않습니다.

사용자가 잘못된 값을 입력할 수도 있고, 존재하지 않는 파일을 열려고 할 수도 있으며, 예상하지 못한 데이터가 프로그램 내부로 들어올 수도 있습니다.

이러한 상황을 처리하지 못하면 프로그램은 즉시 종료(crash)되거나 잘못된 결과를 만들어낼 수 있습니다.

Python의 Exception Handling은 이러한 예외 상황을 감지하고 적절히 대응하여, 프로그램이 안정적으로 동작할 수 있도록 도와주는 핵심 기능입니다.

## 이번 모듈의 핵심 도메인 기술

### 예외 처리 (`try / except`)

예외가 발생하더라도 프로그램이 중단되지 않도록 보호합니다.

어떤 문제가 발생했는지 파악하고,
적절한 복구 로직을 수행할 수 있습니다.

### 예외 발생 (`raise`)

단순히 발생한 에러를 처리하는 것뿐 아니라,

프로그램이 직접 예외를 발생시켜
잘못된 상태를 감지하고 차단하는 방법을 학습합니다.

### 다양한 예외 타입

상황에 따라 서로 다른 예외가 발생합니다.

- ValueError
- TypeError
- ZeroDivisionError
- FileNotFoundError
- KeyError

등 다양한 내장 예외를 이해하고
상황에 맞게 처리하는 방법을 익힙니다.

### 사용자 정의 예외 (Custom Exception)

기본 예외만으로는 의미를 충분히 표현하기 어려울 때

프로그램 목적에 맞는 새로운 예외 클래스를 정의하여
더 명확한 오류 정보를 전달할 수 있습니다.

### 정리 작업 (`finally`)

예외 발생 여부와 상관없이
반드시 실행되어야 하는 작업을 처리합니다.

파일 닫기, 자원 해제, 상태 복구와 같은
정리(cleanup) 로직을 안전하게 수행할 수 있습니다.

## 목표

예외를 단순한 오류가 아닌
프로그램이 스스로 문제를 감지하고 대응하기 위한 기능으로 이해합니다.

이번 모듈에서는

- 예상 가능한 실패를 감지하고
- 적절한 예외를 발생시키며
- 문제 상황을 안전하게 처리하고
- 프로그램의 안정성을 유지하는 방법

을 학습합니다.

이번 모듈의 목표는

**"에러가 발생해도 무너지지 않는 견고한 프로그램을 작성하는 것"** 입니다.
