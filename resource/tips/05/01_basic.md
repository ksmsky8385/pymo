# Module 05 개요

## Object Oriented Design & Polymorphism

이번 모듈에서는 객체지향 프로그래밍의 핵심 개념인

- 추상 클래스 (Abstract Class)
- 상속 (Inheritance)
- 메서드 오버라이딩 (Method Overriding)
- 다형성 (Polymorphism)
- Protocol (Duck Typing)

을 학습합니다.

이번 모듈은 단순히 클래스를 만드는 것이 목적이 아닙니다.

서로 다른 객체들이 동일한 인터페이스를 공유하면서도
각자의 방식으로 동작하는 구조를 설계하는 것이 핵심입니다.

## 이번 모듈의 핵심 도메인 기술

### Abstract Class

객체를 직접 생성하지 못하는 설계용 클래스입니다.

공통 인터페이스를 정의하기 위해 사용합니다.

### Inheritance

부모 클래스의 기능을 자식 클래스가 상속받는 기능입니다.

### Method Overriding

부모 클래스의 메서드를 자식 클래스가 재구현하는 기술입니다.

### Polymorphism

동일한 인터페이스를 사용하지만
객체에 따라 다른 동작을 수행하는 기술입니다.

### Protocol

상속 없이도 동일한 인터페이스를 제공하면
같은 타입처럼 사용할 수 있게 해주는 기능입니다.

## 목표

이번 모듈의 목표는

- 객체지향 설계 이해
- 공통 인터페이스 설계
- 다형성 활용
- 덕 타이핑 이해

입니다.

최종적으로는

"새로운 기능을 추가하더라도 기존 코드를 수정하지 않는 구조"

를 만드는 것이 목표입니다.
