# Module 08 필독 사항

## helper 실행 테스트 개요

Module 08은 Python 버전, 일반 가상환경, 외부 라이브러리, Poetry 가상환경 및 환경변수 파일을 함께 다룹니다.

본 helper는 다음 순서로 실행 테스트를 진행합니다.

1. 모듈 루트의 일반 가상환경 확인 및 필요 시 생성
2. ex0 일반 Python 환경과 가상환경에서 실행
3. ex1 `requirements.txt` 및 Poetry 기반 실행
4. ex2 `python-dotenv`와 `.env` 기반 실행
5. 테스트 산출물 정리 여부 확인

## 정적 검사와 가상환경

flake8과 mypy 정적 검사에서는 이름이 `env`로 끝나는 디렉토리와 그 내부 Python 파일을 검사 대상에서 제외합니다. 가상환경 라이브러리까지 정적 검사하는 문제를 방지하기 위한 helper 전용 처리입니다.

가상환경 외의 과제 Python 파일에는 기존과 동일하게 flake8과 `mypy --strict` 검사가 적용됩니다.

## 일반 가상환경 감지

helper는 `python_module_08` 바로 아래에서 이름이 `env`로 끝나는 디렉토리를 가상환경으로 인식합니다.

예시:

```text
env
.venv
matrix_env
myenv
```

하위 ex 폴더 내부의 가상환경은 감지 대상이 아닙니다.

감지된 디렉토리는 실제로 다음 파일을 포함한 정상적인 Python 가상환경이어야 합니다.

```text
<가상환경>/bin/activate
```

단순 디렉토리의 이름만 `env`로 끝나는 경우에도 helper가 가상환경으로 판단할 수 있으므로 주의하시기 바랍니다.

## Python 3.10 및 pyenv 처리

기존 가상환경이 없으면 현재 Python이 3.10 버전인지 검사합니다.

Python 3.10이 아니면 설치된 pyenv 버전을 확인하고, 선택에 따라 다음 명령과 동일한 설정을 적용합니다.

```shell
pyenv local 3.10.x
```

pyenv가 설치되어 있지 않으면 사용자 로컬 설치, 검사 무시 또는 종료 중 하나를 선택할 수 있습니다. 사용자 로컬 설치를 선택하면 공식 Git 저장소를 다음 경로에 복제합니다.

```text
~/.pyenv
```

사용 중인 로그인 셸이 Bash 또는 Zsh이면 해당 셸 설정 파일에 `PYENV_ROOT`, 실행 경로 및 `pyenv init` 설정도 자동으로 등록합니다. pyenv로 Python을 설치할 때는 시스템에 컴파일 도구와 Python 빌드 의존성이 별도로 필요할 수 있습니다.

이후 모듈 루트에 다음 가상환경을 생성합니다.

```shell
python3 -m venv matrix_env
```

이미 이름이 `env`로 끝나는 가상환경이 있으면 Python 버전 검사와 `pyenv local` 설정을 모두 건너뜁니다. 따라서 기존 가상환경의 Python 버전과 정상 동작 여부는 사용자가 직접 확인해야 합니다.

## ex0 실행 테스트

`construct.py`는 다음 두 환경에서 순서대로 실행됩니다.

1. 가상환경을 활성화하지 않은 일반 Python 환경
2. 감지 또는 생성된 가상환경

두 실행 결과의 Python 경로와 환경 정보가 서로 다르게 출력되는지 확인하시기 바랍니다.

## ex1 requirements.txt 테스트

`loading.py`는 먼저 helper가 선택한 가상환경을 활성화하지 않은 상태에서 실행됩니다. 이 단계에서는 외부 라이브러리가 없을 때 프로그램이 오류를 적절히 처리하고 설치 방법을 안내하는지 확인합니다.

의존성 누락 안내 후 `sys.exit(1)`로 종료하는 것은 이 첫 실행에 한해 정상 테스트 결과로 처리합니다. 다만 Python traceback이 출력되거나 다른 비정상 종료 코드가 반환되면 실행 실패로 처리합니다.

helper 실행 전에 존재하던 가상환경이 감지되면 이미 라이브러리가 준비된 환경으로 간주하여 다음 설치 과정을 생략합니다.

```shell
pip install -r requirements.txt --no-cache-dir
```

기존 가상환경이 없어서 helper가 `matrix_env`를 새로 생성한 경우에는 `requirements.txt`가 존재할 때 라이브러리를 설치합니다.

두 번째 실행에서는 선택한 가상환경을 활성화합니다. 따라서 기존 가상환경을 사용하는 경우 필요한 라이브러리를 미리 설치해 두어야 합니다.

## ex1 Poetry 테스트

Poetry가 설치되어 있지 않으면 설치, 테스트 생략 또는 종료 중 하나를 선택할 수 있습니다.

`ex1/pyproject.toml`이 존재하면 `poetry.lock` 유무에 따라 다음과 같이 동작합니다.

- `poetry.lock` 없음: `poetry install` 실행 후 `poetry run python loading.py` 실행
- `poetry.lock` 있음: `poetry install` 생략 후 바로 `poetry run python loading.py` 실행

`poetry.lock`은 의존성 버전을 기록하는 파일이며 Poetry 가상환경의 존재를 보장하지 않습니다. 락 파일만 있고 가상환경이나 라이브러리가 없는 경우 `poetry run`이 실패할 수 있습니다. 이 경우 직접 `poetry install`을 실행하거나 락 파일을 제거한 뒤 helper 테스트를 다시 실행하시기 바랍니다.

Poetry 가상환경은 다음 명령으로 확인할 수 있습니다.

```shell
cd ex1
poetry env list --full-path
poetry env info --path
```

로그아웃 또는 `poetry.lock` 삭제만으로 Poetry 가상환경이 자동 삭제되지는 않습니다.

## ex2 .env 테스트

helper는 선택된 일반 가상환경에서 `python-dotenv`를 확인하고, 설치되어 있지 않으면 자동으로 설치합니다.

`ex2/.env`가 이미 존재하면 다음 중 하나를 선택합니다.

1. 기존 `.env` 삭제
2. 기존 `.env`를 유지하고 실행
3. 테스트 종료

기존 `.env`를 유지하면 해당 파일로 `oracle.py`를 한 번 실행하고, `.env.example`을 `.env`로 복사하는 과정과 두 번째 실행은 생략합니다.

기존 `.env`를 삭제하거나 처음부터 `.env`가 없으면 `oracle.py`를 실행한 뒤 다음 과정으로 `.env.example` 적용 결과를 다시 확인합니다.

```shell
cp .env.example .env
python3 oracle.py
```

`.env`에는 API 키와 같은 민감정보가 포함될 수 있으므로 Git 저장소에 커밋하지 않도록 `.gitignore` 설정도 확인하시기 바랍니다.

## 테스트 산출물 정리 주의사항

Module 08 테스트 종료 시 산출물 삭제 여부를 묻습니다. 입력 없이 엔터를 누르면 삭제가 진행되는 것이 기본 동작입니다.

삭제를 선택하면 helper는 다음 항목을 정리합니다.

- 감지했거나 생성한 일반 가상환경
- `.python-version`
- 테스트 중 생성된 PNG 파일
- `ex1/poetry.lock`
- `ex2/.env`
- 이름이 `matrix-loading-`으로 시작하는 ex1 Poetry 가상환경

helper 실행 전부터 사용하던 가상환경도 삭제 대상이 될 수 있습니다. 가상환경, 락 파일 또는 `.env`를 보존하려면 정리 질문에서 반드시 `n`을 입력하시기 바랍니다.
