# Module 08 개요

## Python 실행 환경과 설정 관리

이번 모듈에서는 실제 프로젝트에서 자주 사용하는 실행 환경 관리 방법을 학습합니다.

핵심 개념은 다음과 같습니다.

- Virtual Environment
- pip와 `requirements.txt`
- Poetry와 `pyproject.toml`
- 외부 라이브러리 확인 및 예외 처리
- 환경변수와 `.env`
- 민감정보 관리

## .gitignore

가상환경은 설치된 Python과 라이브러리를 포함하므로 크기가 크고 다른 환경에서 그대로 사용할 수 없습니다.

가상환경을 만들기 전에 프로젝트 루트의 `.gitignore`에 사용할 디렉토리 이름을 추가하는 것이 좋습니다.

```gitignore
matrix_env/
.venv/
env/
```

Poetry가 프로젝트 내부에 가상환경을 생성하도록 설정된 경우에도 일반적으로 `.venv/`가 사용됩니다.

가상환경은 Git으로 공유하지 않고 `requirements.txt`, `pyproject.toml`, `poetry.lock`과 같은 의존성 파일을 이용해 다시 생성합니다.

`.gitignore`에 추가하기 전에 이미 Git이 추적한 파일은 자동으로 추적 해제되지 않으므로 가상환경 생성 직후 `git status`를 확인하는 습관이 중요합니다.

## 목표

같은 Python 코드라도 실행 환경에 따라

- Python 실행 파일
- 설치된 라이브러리
- 라이브러리 버전
- 애플리케이션 설정

이 달라질 수 있다는 점을 이해하는 것이 목표입니다.

이번 모듈의 목표는

- 프로젝트별 독립 실행 환경 구성
- 의존성 파일을 이용한 라이브러리 관리
- 환경변수를 이용한 실행 설정 분리
- `.gitignore`를 이용한 가상환경과 `.env`의 실수로 인한 커밋 방지
- API 키와 같은 민감정보를 코드에서 분리하여 관리

하는 것입니다.

최종적으로는 프로젝트마다 독립된 환경과 설정을 사용하는 방법을 익히게 됩니다.
